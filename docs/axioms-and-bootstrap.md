# Axioms and the trace bootstrap

This note states the axioms a `KAlgebra` satisfies, how a concrete algebra is
checked against them, and the sense in which the orthonormality axiom determines
the trace — the property the realisations in `src/cone/` rely on.

## The axioms

A K_𝖖-algebra is an associative algebra `A_𝖖` over `Z[𝖖^±]`, free as a
`Z[𝖖^±]`-module, carrying:

1. a **bar involution** — an antimultiplicative `Z`-algebra involution sending
   `𝖖 ↦ 𝖖⁻¹`;
2. a **canonical basis** `{L_a}` of bar-invariant elements, containing the unit;
3. an algebra **automorphism `ρ`** acting as a permutation `L_a ↦ L_{ρ(a)}` of
   the canonical basis;
4. a **`ρ²`-twisted trace** `Tr : A_𝖖 → Z((𝖖))`, linear over `Z[𝖖^±]`, whose
   pairing satisfies
   ```
   I_{a,b} = Tr(L_{ρ(a)} L_b) = Tr(L_a L_{ρ⁻¹(b)}) = δ_{a,b} + O(𝖖).
   ```
   The trace is determined only up to an overall factor `1 + O(𝖖)`.

A **flavoured** K_𝖖-algebra is enriched to an algebra over `Z[𝖖^±] ⊗ R_{G_f}`
for a reductive flavour group `G_f`: the canonical basis additionally contains
the irreducible characters `χ_r ∈ R_{G_f}`; `ρ` is twisted-linear with
`ρ(χ_r) = χ_{r^∨}`; and the trace takes values in `Z((𝖖)) ⊗ R_{G_f}`, with the
identity-character part of `I_{a,b}` equal to `δ_{a,b} + O(𝖖)`.

## Checking a concrete algebra

A concrete algebra subclasses `KAlgebra` (`src/core/kalgebra.py`) and supplies
the primitives `coefficient_ring`, `identity`, `multiply`, `rho` /
`rho_inverse`, and `trace` (with a flavour-lift coordinate when flavoured). The
base class derives the bilinear product, the pairing `I_{a,b}`, the
`Z`-form / `R`-form views, and a battery of verifiers:

| structure | verifiers |
|---|---|
| associative unital algebra | `verify_unit_law`, `verify_associativity` |
| bar involution | `verify_bar_involution` |
| canonical basis | `verify_identity_in_basis` |
| automorphism `ρ` | `verify_rho_is_automorphism`, `verify_rho_inverse`, `verify_rho_fixes_identity` |
| `ρ²`-twisted trace | `verify_rho_twisted_trace`, `verify_trace_pairing_faces` |
| orthonormality | `verify_orthonormality` |

Arithmetic is exact (`Z[𝖖^±]` / `R[𝖖^±]`); truncation to `O(𝖖^K)` occurs only
where a trace is read out. A passing `run_tests.py` certifies the axioms on every
label exercised.

## The orthonormality axiom determines the trace

Orthonormality is a strong constraint: together with `ρ²`-twisted cyclicity it
fixes the trace from the single value `Tr 1`. The pentagon `K_𝖖([A_1,A_2])`
(`PentagonSampleKAlgebra`) is the smallest case. It is generated by `L_i`,
`i ∈ Z/5`, with

```
L_{i+1} L_i      = 𝖖² L_i L_{i+1},
L_{i+1} L_{i-1}  = 1 + 𝖖 L_i,
L_{i-1} L_{i+1}  = 1 + 𝖖⁻¹ L_i,
```

`ρ(L_i) = L_{i+2}`, and the remaining canonical elements
`L_{i;a,b} = 𝖖^{ab} L_i^a L_{i+1}^b`. Cyclicity reduces every trace to `Tr 1` and
`Tr L_i`, with

```
Tr L_i^n = 𝖖^{1-2n} Tr L_i^{n-1} + 𝖖^{2-2n} Tr L_i^{n-2}.
```

The axiom requires `Tr L_i^n ∈ δ_{n,0} + 𝖖 Z[[𝖖]]` — in particular no negative
powers of `𝖖` — and each step of the recursion then pins the ratio
`Tr L_i / Tr 1` to one further order, reaching `O(𝖖^{n²})` at step `n`. The
limiting ratio is the continued fraction

```
1 − 𝖖 (Tr L_i / Tr 1) = 1 + 𝖖²/(1 + 𝖖⁴/(1 + 𝖖⁶/(1 + ⋯))),
```

a ratio of Rogers–Ramanujan (`M(2,5)`) characters, with `Tr 1` the corresponding
vacuum character. The trace is thus a consequence of `Tr 1` and the axiom alone.
The `U_𝖖(𝔰𝔩₂)` algebra (SQED₂) and `K_𝖖([A_1,D_3])` are fixed the same way.

### In the code

The `ConeKAlgebra` realisations (`src/cone/`) carry out this determination:

1. **Reduction** — `ρ²`-twisted cyclicity over the cone data reduces `Tr(L_a)`,
   for any label, to a finite set of elementary seeds (`Tr 1` and the
   single-generator traces).
2. **Seeds** — each seed is fixed either by a known closed-form character (for
   example the `M(2,2k+3)` characters in `minimal_model_characters`) or by the
   orthonormality bootstrap (`trace_uniqueness_proofs` and the per-flavour
   drivers), solved order by order from the axiom.
3. **`Tr 1`** — computed as the exact Nahm sum on the BPS spectrum
   (`vacuum_nahm.py`), to arbitrary `𝖖`-order.

The bootstrap proceeds order by order with no truncation, so the traces are exact
and extend to any `𝖖^K`; `test_cones.py` checks this by continuing past any fixed
order. Run with only this package on the import path, a passing `test_cones.py`
exhibits the traces as consequences of the axioms rather than of an external
(BPS- or RG-based) construction.

## Two presentations of one algebra

`KAlgebraIso` (`src/core/kalgebra_iso.py`) certifies that two presentations are
the same K_𝖖-algebra, via a bijective label map checked for the unit,
round-trip, multiplicativity, `ρ`-equivariance, and trace-equivariance in both
directions. `src/iso/` certifies three sample-to-cone identifications:

| algebra | sample | cone |
|---|---|---|
| pentagon `K_𝖖([A_1,A_2])` | `PentagonSampleKAlgebra` | `FinitePentagonKAlgebra` |
| SQED₁ `[A_1,A_1]` | `SQED1SampleKAlgebra` | `U1SquareKAlg` |
| SQED₂ `[A_1,D_2]` | `SQED2SampleKAlgebra` | `U1A1D2ConeKAlgebra` |

verified by `tests/test_sample_cone_iso.py`.
